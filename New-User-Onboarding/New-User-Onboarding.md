# Issue Title: New User Onboarding & Account Setup in Active Directory

# Scenario
A new employee joins the company and needs an account created, access configured, and workstation prepared before their first day.

# Environment
- Windows 10/11 company PC
- Active Directory Users and Computers (ADUC)
- User groups: Domain Users, Department Security Groups

# Tasks Performed (Step-by-Step)
1. Received onboarding ticket with user details (name, department, manager)
2. Verified request approval from HR/manager
3. Opened **Active Directory Users and Computers**
4. Created new user under correct OU
5. Assigned temporary secure password
6. Added user to required security groups:
   - Domain Users
   - Department access group
   - Distribution or security groups
7. Configured password settings:
   - User must change password at next logon
8. Logged in with new user account to validate credentials
9. Confirmed access to required resources:
   - Shared network drives
   - Email mailbox
   - Internal websites
10. Updated ticket with completion notes

# Outcome
User account successfully created, permissions granted, and user was able to login and access required tools on first day.

# Skills Demonstrated
- Active Directory user management
- Security group and permission assignment
- Identity verification and onboarding workflow
- ITIL-style ticket handling / documentation

# Recommendations
- Standardize access profiles per department
- Maintain onboarding checklist
- Use naming conventions for AD accounts
